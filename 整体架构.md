# 个性化旅游系统 — 整体架构

## 一、技术栈总览

| 层级 | 技术 | 说明 |
|------|------|------|
| **后端** | Flask 2.x | 轻量级Web框架，Blueprint模块化路由 |
| **前端** | Vue 3 (CDN) | Composition API，响应式数据绑定 |
| **模板** | Jinja2 | 服务端渲染HTML |
| **数据库** | MySQL 8.x (预留) | 关系型数据库，替代Pickle文件存储 |
| **地图** | 高德地图 JS API 2.0 | POI搜索/路线规划 |
| **数据存储** | Pickle文件 (当前) | 适合单机Demo场景 |
| **图标** | Font Awesome 6.4 | UI图标库 |

## 二、核心架构

### 2.1 分层架构

```
表现层 (Vue 3 模板)        ← web_app/templates/
         ↓ fetch() JSON
路由控制层 (Blueprint)     ← backend/routes/
         ↓
业务逻辑层 (Service)       ← services/
         ↓
数据访问层 (ORM, 预留)     ← models/ + db/
         ↓
持久层                     ← Pickle(当前) / MySQL(目标)
```

### 2.2 关键设计决策

- **前后端混合渲染**：Flask Jinja2 渲染初始页面 → Vue 3 接管交互逻辑
- **Vue 自定义定界符**：`{$ $}` 避免与 Jinja2 `{{ }}` 冲突
- **双模式 API**：同时支持表单提交和 JSON 请求
- **单例服务模式**：Service 模块级实例化，全局共享
- **MySQL 迁移预留**：已构建完整 ORM 模型和数据库初始化架构

## 四、数据流

```
Vue 前端 → fetch() JSON API → Flask Route → Service 层 → 数据操作 → 响应 JSON
```

## 六、MySQL 迁移路径

1. **安装依赖**：`pip install flask-sqlalchemy pymysql`
2. **创建数据库**：`CREATE DATABASE travel_system DEFAULT CHARSET utf8mb4;`
3. **配置连接**：修改 `config/settings.py` 中的 `DATABASE_URL`
4. **启用 ORM**：取消注释相关代码
5. **重写 Service**：使用 `db.session.query(Model)` 替代 dict 操作
6. **数据迁移**：从 Pickle 导入到 MySQL
7. **清理旧文件**：移除 `utils/storage.py` 和 `data/db/*.pkl`
