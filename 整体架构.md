# 个性化旅游系统 — 整体架构

## 一、技术栈总览

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **后端框架** | Flask | 2.0+ | 轻量级、Blueprint 模块化路由、Jinja2 模板引擎 |
| **前端框架** | Vue | 3.x (CDN) | Composition API、响应式数据绑定、无需构建工具 |
| **模板引擎** | Jinja2 | 3.0+ | 服务端渲染 HTML 模板，与 Flask 深度集成 |
| **数据库** | MySQL | 8.0+ (预留) | 关系型数据库，支持事务 ACID，适合结构化数据 |
| **地图服务** | 高德地图 | JS API 2.0 | POI 搜索、多种路线规划、地理编码 |
| **数据存储** | Pickle | Python 内置 | 适合单机 Demo 场景，简单易用 |
| **密码安全** | SHA256 | 内置 | 单向加密，密码不可逆 |
| **图标库** | Font Awesome | 6.4 | 统一的 UI 图标体系，丰富的图标资源 |

### 2.3 关键设计决策

1. **前后端混合渲染**
   - Flask Jinja2 渲染初始 HTML 页面
   - Vue 3 在浏览器中接管交互逻辑
   - 优势：首屏加载快，交互流畅

2. **Vue 自定义定界符**
   - 使用 `{$ $}` 替代默认的 `{{ }}`
   - 避免与 Jinja2 模板语法冲突
   - 实现：`delimiters: ['{$', '$}']`

3. **双模式 API 兼容**
   - 后端接口同时支持表单提交和 JSON 请求
   - 平滑迁移，兼容性好
   - 实现：检查 `Content-Type` 动态处理

4. **单例服务模式**
   - Service 在模块级别实例化，全局共享
   - 避免重复初始化，提高性能
   - 实现：模块级变量 `user_service = UserService()`

5. **MySQL 迁移预留**
   - 已构建完整 ORM 模型和数据库初始化架构
   - 取消注释即可完成 Pickle → MySQL 迁移
   - 包含完整的 DDL 语句和 ER 图

### 4.2 DDL 建表语句

```sql
-- 用户表
CREATE TABLE users (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    username    VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    email       VARCHAR(100) NOT NULL UNIQUE COMMENT '邮箱',
    password_hash CHAR(64) NOT NULL COMMENT 'SHA256 密码哈希',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 日记表
CREATE TABLE diaries (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT NOT NULL COMMENT '所属用户ID',
    title       VARCHAR(200) NOT NULL COMMENT '日记标题',
    content     TEXT COMMENT '日记内容',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
CREATE INDEX idx_diaries_user_id ON diaries(user_id);
```

### 4.3 ORM 模型代码

```python
# models/user_model.py
from db import db
from datetime import datetime

class User(db.Model):
    __tablename__ = 'users'

    id = db.Column(db.Integer, primary_key=True, autoincrement=True)
    username = db.Column(db.String(50), unique=True, nullable=False)
    email = db.Column(db.String(100), unique=True, nullable=False)
    password_hash = db.Column(db.String(64), nullable=False)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

    # 关系：一个用户拥有多篇日记
    diaries = db.relationship('Diary', backref='user', lazy='dynamic', cascade='all, delete-orphan')

# models/diary_model.py
class Diary(db.Model):
    __tablename__ = 'diaries'

    id = db.Column(db.Integer, primary_key=True, autoincrement=True)
    user_id = db.Column(db.Integer, db.ForeignKey('users.id'), nullable=False)
    title = db.Column(db.String(200), nullable=False)
    content = db.Column(db.Text)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

## 六、关键代码设计模式

### 6.1 单例服务模式

所有 Service 在模块级别实例化，整个应用共享一个实例：

```python
# services/user_service.py
class UserService:
    def __init__(self):
        self.db = db  # 注入 SQLAlchemy 实例

user_service = UserService()  # 模块级单例

# backend/routes/auth.py
from services.user_service import user_service  # 直接引用
user_service.verify_user(username, password)
```

### 6.2 Flask Blueprint 模块化

```python
# backend/app.py - 注册中心
def register_blueprints(app):
    app.register_blueprint(main_bp)       # /
    app.register_blueprint(auth_bp)       # /login, /register
    app.register_blueprint(diary_bp)      # /api/diary (url_prefix='/api')
    app.register_blueprint(recommend_bp)  # /api/spots, /api/food
    app.register_blueprint(rating_bp)     # /api/rating
```

### 6.3 数据库初始化模式

```python
# db.py - SQLAlchemy 引擎 + 连接池配置
from flask_sqlalchemy import SQLAlchemy

db = SQLAlchemy()

def init_db(app):
    db.init_app(app)

    with app.app_context():
        db.create_all()  # 自动根据 Model 创建所有表
        print("✅ 数据库表已就绪")

# backend/app.py - 应用工厂中初始化
def create_app():
    app = Flask(__name__)
    app.config.from_object(Config)  # 包含 SQLALCHEMY_DATABASE_URI 等
    init_db(app)                    # 初始化数据库
    register_blueprints(app)        # 注册蓝图
    return app
```

## 八、MySQL 迁移实施路线图

### Phase 1: 基础设施搭建
1. 安装依赖：`pip install flask-sqlalchemy pymysql`
2. 创建 `models/` 目录，编写 User / Diary ORM 模型
3. 创建 `db/db.py`，配置 SQLAlchemy + MySQL 连接串
4. 更新 `config/settings.py`，添加 `DATABASE_URL` 配置项
5. 在 `backend/app.py` 中调用 `init_db(app)` 自动建表

### Phase 2: Service 层改造
6. 重写 `user_service.py`：`dict 操作 → db.session.query(User)`
7. 重写 `diary_service.py`：`dict 操作 → db.session.query(Diary)`，移除 `_next_ids`
8. 其他 Service 不涉及持久化改动，保持原样

### Phase 3: 数据迁移
9. 编写迁移脚本：读取旧 `.pkl` 文件 → 批量 INSERT 到 MySQL
10. 验证数据一致性（用户数、日记数、ID 对应关系）

### Phase 4: 清理收尾
11. 移除 `utils/storage.py` 或标记为 @deprecated
12. 删除 `data/db/*.pkl` 文件
13. 更新 requirements.txt，移除不需要的依赖

## 九、性能优化策略

1. **数据库连接池**：通过 `SQLALCHEMY_ENGINE_OPTIONS` 配置连接池大小，避免频繁创建/销毁连接

2. **缓存机制**：对于静态推荐数据，可考虑使用 Redis 缓存，减少数据库查询

3. **前端优化**：
   - Vue 3 响应式数据合理使用，避免不必要的重新渲染
   - 图片懒加载，减少首屏加载时间
   - 高德地图按需加载，避免影响页面加载速度

4. **后端优化**：
   - 使用 `gzip` 压缩响应数据
   - 合理使用 `@cache` 装饰器缓存热点数据
   - 批量操作减少数据库交互次数

## 十、安全考虑

1. **密码安全**：使用 SHA256 哈希存储密码，禁止明文存储

2. **SQL 注入防护**：使用 SQLAlchemy ORM，避免直接拼接 SQL 语句

3. **XSS 防护**：前端输入验证，后端 HTML 转义

4. **CSRF 防护**：Flask 内置 CSRF 保护，确保表单提交安全

5. **Session 安全**：使用安全的 Session 存储，设置合理的过期时间

6. **API 访问控制**：对敏感 API 端点实施 `@login_required` 保护
