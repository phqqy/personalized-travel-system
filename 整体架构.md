# 个性化旅游系统 — 整体架构

## 一、技术栈总览

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **后端框架** | Flask | 2.0+ | 轻量级、Blueprint 模块化路由、Jinja2 模板引擎 |
| **前端框架** | Vue | 3.x (CDN) | Composition API、响应式数据绑定、无需构建工具 |
| **模板引擎** | Jinja2 | 3.0+ | 服务端渲染 HTML 模板，与 Flask 深度集成 |
| **数据库** | MySQL | 8.0+ (预留) | 关系型数据库，支持事务 ACID，适合结构化数据 |
| **地图服务** | 高德地图 | JS API 2.0 | POI 搜索、多种路线规划、地理编码 |
| **数据存储** | Pickle | Python 内置 | 适合单机 Demo 场景，简单易用 |
| **密码安全** | SHA256 | 内置 | 单向加密，密码不可逆 |
| **图标库** | Font Awesome | 6.4 | 统一的 UI 图标体系，丰富的图标资源 |

### 2.2 各层职责说明

| 层级 | 职责 | 关键文件 | 技术特点 |
|------|------|----------|----------|
| **表现层** | 页面展示、用户交互、弹窗提示 | 8 个 Vue 3 HTML 模板 | Composition API、响应式数据 |
| **路由层** | HTTP 请求分发、参数校验、调用 Service | Blueprint 路由模块 | 模块化路由、URL 前缀管理 |
| **装饰器层** | 横切关注点：登录校验 | `@login_required` | 统一的登录状态检查 |
| **业务层** | 核心业务逻辑、数据处理规则 | 4 个 Service 单例 | 单例模式、业务逻辑集中 |
| **数据访问层** | ORM 模型定义、表结构映射、查询封装 | User / Diary Model | 类型安全、自动建表 |
| **数据库初始化层** | 引擎创建、连接池配置、建表迁移 | db.py | 连接池管理、自动迁移 |
| **配置层** | 全局参数集中管理 | Config 类 | 统一配置、环境变量支持 |
| **持久层** | 数据存储 | MySQL / Pickle | 关系型存储、文件存储 |

### 2.3 关键设计决策

1. **前后端混合渲染**
   - Flask Jinja2 渲染初始 HTML 页面
   - Vue 3 在浏览器中接管交互逻辑
   - 优势：首屏加载快，交互流畅

2. **Vue 自定义定界符**
   - 使用 `{$ $}` 替代默认的 `{{ }}`
   - 避免与 Jinja2 模板语法冲突
   - 实现：`delimiters: ['{$', '$}']`

3. **双模式 API 兼容**
   - 后端接口同时支持表单提交和 JSON 请求
   - 平滑迁移，兼容性好
   - 实现：检查 `Content-Type` 动态处理

4. **单例服务模式**
   - Service 在模块级别实例化，全局共享
   - 避免重复初始化，提高性能
   - 实现：模块级变量 `user_service = UserService()`

5. **MySQL 迁移预留**
   - 已构建完整 ORM 模型和数据库初始化架构
   - 取消注释即可完成 Pickle → MySQL 迁移
   - 包含完整的 DDL 语句和 ER 图

## 四、总结

个性化旅游系统采用 **Flask + Vue 3** 全栈架构，结合 MySQL 数据库预留，实现了一个功能完整、架构清晰的旅游推荐与导航系统。系统具有良好的扩展性和可维护性，为用户提供了个性化的旅游体验。

通过模块化的设计和分层架构，系统实现了前后端的高效协作，同时预留了 MySQL 迁移路径，为未来的功能扩展和性能优化打下了坚实的基础。
